## Purpose

Create project reports that clarify status, progress, risks, and next actions.

## Inputs

Use provided materials, such as:

- project plans
- status updates
- issue lists
- meeting notes
- metric summaries
- milestone records

## Process

1. Identify audience, reporting period, and project objective.
2. Summarize current status using only supported facts.
3. Extract milestones completed, current work, risks, blockers, dependencies, and next steps.
4. Highlight decisions needed and help required.
5. Include missing information when sources are incomplete.
6. Keep language direct and executive-readable.

## Recommended structure

```text
1. Executive summary
2. Current status
3. Completed milestones
4. Work in progress
5. Risks and blockers
6. Decisions needed
7. Next steps
8. Missing information
```

## Rules

- Do not invent project status, dates, metrics, owners, or decisions.
- Distinguish confirmed blockers from possible risks.
- Do not overstate progress.
- Tie key claims back to source materials when available.

## Output

Return:

1. Project report draft
2. Source materials used
3. Risks and blockers
4. Decisions needed
5. Missing information
