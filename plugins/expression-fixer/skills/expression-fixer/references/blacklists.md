# 禁用词汇、句式与过度判断参考清单

本文件是 expression-fixer skill 的详细参考。按检测层级组织，每类包含关键词表、检测规则和修正示例。

---

## 一、元语言关键词（第一层 · 必须修正）

元语言指文本在讨论"怎么写/怎么报"这件事本身，而不是在写内容。这类问题最隐蔽也最致命，因为它直接暴露了"这份文档经过讨论和修改"的过程。

### 1.1 元语言动词

这些动词在工作文档中出现时，大概率是在描述"写稿/改稿"动作而非业务动作：

| 禁用词 | 替代方案 | 说明 |
|--------|----------|------|
| 写成 | 认定为、等同于、视为 | "写成"是编辑动作 |
| 写为 | 定义为、归类为 | 同上 |
| 改为 | 调整为、变更为 | "改"暗示修稿 |
| 表述为 | 定义为、界定为、称为 | "表述"在讨论文字 |
| 表述压回 | 定位收回到 | 在改稿 |
| 上报 | 跟踪、纳入管理 | "上报"暴露汇报链条 |
| 汇报 | 同步、通报、更新 | 同上（但"向XX汇报"作为动作描述除外） |

### 1.2 元语言名词

这些名词直接指向文档本身或写作过程：

| 禁用词 | 替代方案 | 说明 |
|--------|----------|------|
| 周报 | 跟踪、本周 | 除非确实在讨论周报制度 |
| 日报 | 每日跟踪 | 同上 |
| 文稿 | （删除或重写整句） | 在讨论文档 |
| 口径 | 标准、定义、范围 | "口径"暴露内部对账 |
| 提法 | 定义、分类 | 在讨论措辞 |
| 说法 | 结论、判断 | 同上 |
| 措辞 | （删除或重写整句） | 直接讨论文字 |

### 1.3 元语言句式

| 禁用句式 | 替代 | 说明 |
|----------|------|------|
| 不能把X写成Y | 不能把X等同于Y / X不等于Y | 去掉"写" |
| 在报告中体现 | 纳入跟踪 / 单独列出 | 去掉"报告" |
| 式过满表述 | 等超前结论 | 去掉"表述" |
| 混用一个口径上报 | 混在一起跟踪 | 去掉"口径""上报" |
| 后续周报中需要 | 后续需要 / 下一步需要 | 去掉"周报" |

### 1.4 内部黑话（读者不知道的内部简称或对账数字）

| 类型 | 示例 | 问题 | 修正 |
|------|------|------|------|
| 内部编号 | 补齐 A 类/B 类任务缺口 | 高层看不懂 | 补齐未申报任务和识别不足任务的缺口 |
| 缩写 | 目标里的第 3 项关键结果 | 读者可能不知道具体指什么 | 展开说明具体内容 |
| 内部代号 | "项目代号A"第二阶段 | 如果读者不了解这个代号 | 首次出现时加括号说明 |

## 二、来源暴露关键词（第一层 · 必须修正）

直接引用信息来源，让读者知道了不该知道的"信息从哪来"。

### 2.1 会议相关

| 禁用词 | 替代方案 |
|--------|----------|
| 会议纪要 | （删除来源，直接陈述结论） |
| 会上提到 | （直接说结论） |
| XX会议决定 | 已决定 / 方案已确定（除非会议本身是重要信息） |
| 评审会结论 | 评审结论 |

### 2.2 沟通相关

| 禁用词 | 替代方案 |
|--------|----------|
| 根据XX的邮件 | （直接陈述内容） |
| XX说过 / XX提到 | （直接陈述内容，或用"经确认"） |
| 聊天记录显示 | （删除来源） |
| 群里说 | 经了解 / 经确认 |
| 私下沟通后了解到 | 经了解 |

### 2.3 文档相关

| 禁用词 | 替代方案 |
|--------|----------|
| 需求文档中写的 | 需求定义为 |
| 规划文档已明确 | 已明确 / 规划如下 |
| 参照上周的表格 | （直接引用数据） |

**判断原则**：如果去掉来源后信息仍然完整，就删；如果来源本身是关键信息（比如"客户在验收报告中提出"），则保留。

## 三、搭配不当词汇表（第二层）

### 3.1 万能动词黑名单

以下动词本身不是错，但容易被滥用——与不恰当的宾语硬搭，或在同一篇文档中反复出现。**同一篇出现超过 2 次时检查每处搭配是否合理。**

| 万能动词 | 常见硬搭（不自然） | 更准确的替代 |
|----------|---------------------|--------------|
| 打通 | 打通能力、打通认知 | 建立能力、统一认知 |
| 拉齐 | 拉齐水平、拉齐进度 | 统一水平、同步进度 |
| 收敛 | 收敛能力、收敛方向 | 明确方向、聚焦能力 |
| 收紧 | 收紧闭环、收紧效果 | 完成闭环、验证效果 |
| 闭环 | 闭环问题（作动词用） | 解决问题、关闭问题 |
| 对齐 | 对齐思路、对齐目标 | 统一思路、明确目标 |
| 赋能 | 赋能团队、赋能业务 | 支撑团队、支持业务 |
| 沉淀 | 沉淀能力、沉淀经验 | 积累经验、形成能力 |
| 拉通 | 拉通流程、拉通信息 | 打通流程、共享信息 |
| 穿透 | 穿透数据、穿透管理 | 追踪数据、深入管理 |
| 透传 | 透传需求、透传压力 | 传达需求、传导压力 |
| 触达 | 触达用户、触达目标 | 覆盖用户、达成目标 |

### 3.2 名词黑话黑名单

| 黑话 | 问题 | 更准确的替代 |
|------|------|--------------|
| 抓手 | 万能名词 | 切入点、工具、手段、措施 |
| 颗粒度 | 技术术语滥用 | 精细程度、细化程度、详细程度 |
| 组合拳 | 过度比喻 | 综合措施、多项措施 |
| 护城河 | 过度比喻 | 竞争优势、核心壁垒 |
| 底座 | 过度比喻 | 基础、平台、底层 |
| 飞轮 | 过度比喻 | 正循环、良性循环 |

### 3.3 "更能/最能"系列搭配问题

"更能""最能"后面接的动词要注意，很多搭配不自然：

| 不自然 | 自然 |
|--------|------|
| 更能确认 | 可以确认 / 更加确定 |
| 更能保障 | 更好地保障 |
| 最能体现 | 最能说明 / 最直接地体现 |

## 四、句式重复骨架清单（第三层）

### 4.1 "不是X，而是Y"系列

全文出现次数上限：3 次。超过即削减非必要的使用。

保留条件：语义上确实存在"常见误解 vs 实际情况"的对比。
削减条件：只是为了强调后半句，前半句是为了铺垫而硬造的否定。

| 变体 | 保留/削减 | 替代方案 |
|------|-----------|----------|
| 不是A，而是B | 看语境 | 真正的X在于B / 核心是B |
| 与其A，不如B | 通常削减 | 下一步B |
| 不是说A不重要，而是B更关键 | 通常削减 | B是当前优先级 |
| 关键不在A，在B | 看语境 | 重点是B |

### 4.2 "不能只X，要Y"系列

全文出现次数上限：2 次。

| 变体 | 替代方案 |
|------|----------|
| 不能只看X，要看Y | 除了X，还需关注Y |
| 不能停留在X，要做到Y | 从X进一步到Y |
| 不能满足于X | X只是起点 |

### 4.3 "从A到B"系列

全文出现次数上限：2 次。

| 变体 | 替代方案 |
|------|----------|
| 从A收紧到B | 聚焦到B |
| 从A升级到B | 目标是B |
| 从A转向B | 重心移到B |

### 4.4 排比/递进系列

"一是…二是…三是…"全文出现次数上限：2 次。三段式列举是典型的报告腔。

替代方案：
- 改为自然段落叙述
- 改为两项（去掉最弱的一项）
- 改为四项（打破三段式节奏）

## 五、语气问题检测词（第二层）

### 5.1 说教语气

| 关键词/句式 | 替代 |
|-------------|------|
| 与其…不如… | 下一步… |
| 不应该再… | 调整为… |
| 必须认识到 | 实际情况是 |
| 要清醒地看到 | 值得注意的是（或直接陈述） |
| 各方需要意识到 | （删除，直接说事） |

### 5.2 辩解语气

| 关键词/句式 | 替代 |
|-------------|------|
| 并非…而是因为… | （直接说原因） |
| 不是因为我们没做，是因为… | （删除辩解，说下一步） |
| 当前不应把重心放在"为什么没做到" | 路径已明确，当前重心是… |
| 客观来说… | （删除，直接陈述） |

### 5.3 下指令语气

| 关键词/句式 | 替代 |
|-------------|------|
| 务必 | 计划 / 目标是 |
| 确保 | 目标是 / 争取 |
| 要求各方 | 各方计划 / 协同推进 |
| 需要立即 | 优先处理 |
| 必须在XX前完成 | 计划在XX前完成 |

**语气判断原则**：写周报的人通常不是下指令的人。如果作者没有指令权限，用指令语气会让读者反感。

## 六、过度判断词汇表（额外检测项）

### 6.1 完成度过度判断

| 过度用词 | 什么时候能用 | 不能用时的替代 |
|----------|-------------|----------------|
| 已完成 | 所有子项全部做完且验证通过 | 主体完成，待XX / 已完成X项，剩余Y项 |
| 已闭环 | 问题已解决且不会复发 | 已有方案，执行中 / 初步闭环，持续观察 |
| 已打通 | 端到端流程可以跑通 | 已对接，待联调 / 主流程已通，分支待验证 |
| 已上线 | 功能在生产环境可用 | 已部署到测试环境 / 已上线灰度 |
| 已落地 | 方案在实际业务中产生效果 | 已启动执行 / 已在X场景试点 |

### 6.2 程度过度判断

| 过度用词 | 替代 |
|----------|------|
| 全面完成 | 完成主要任务 / 完成X项（共Y项） |
| 圆满完成 | 按计划完成 |
| 成功实现 | 实现了 / 达成了 |
| 显著提升 | 提升了X%（用数据说话） |
| 大幅改善 | 改善了，具体表现为… |
| 取得突破 | 取得进展 / 解决了XX问题 |
| 效果明显 | 具体效果：… |

### 6.3 判断准确度自检

对每个判断词，问三个问题：
1. 有没有数据或事实支撑这个判断？
2. 如果下周被问"怎么验证"，答得上来吗？
3. 如果事后发现没达到，会不会被打脸？

三个问题中有一个答"不确定"，就降一级用词。

## 七、信息密度控制规则（第三层）

### 7.1 并列成分上限

| 并列数量 | 处理方式 |
|----------|----------|
| ≤3 个 | 正常，不需处理 |
| 4 个 | 可接受，但检查是否都必要 |
| 5-6 个 | 用分号、"以及"、或逻辑分组断句 |
| ≥7 个 | 必须拆句或用层级结构 |

### 7.2 分组断句技巧

**分组法**：按逻辑将并列项归为 2-3 组

原文：业务入口、输入质量、人工复核、监控度量和承载条件还没有一起闭合
修正：前端环节（业务入口、输入质量）和后端环节（人工复核、监控度量、承载条件）尚未完全打通

**加副词断句法**：插入"逐一""分别""依次"等词

原文：围绕三个样板收紧业务入口、数据来源、人工复核和度量边界
修正：围绕三个样板，逐一明确业务入口、数据来源、人工复核和度量边界

**拆句法**：直接拆成两句

原文：首个质量判断、数据口径、人工复核和度量闭环未收紧
修正：首个质量判断和数据口径尚未统一。人工复核和度量方式也有待明确。

### 7.3 同义重复检测

同一句话中不应出现两个意思相近的词做不同成分：

| 重复 | 修正 |
|------|------|
| 目标效果没有**写清**…导致…无法**写清** | 标题用"明确"，正文用"理清" |
| **完善**流程并**优化**机制 | 选一个：完善流程和机制 |
| **确认**需求并**明确**方向 | 选一个：明确需求和方向 |
