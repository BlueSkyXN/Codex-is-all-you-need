# 项目能力表 XLSX 规则

用户明确要求 `XLSX`、`Excel`、`工作簿`、`表格阅读版`、`方便阅读的 Excel 版`、`完全汉化 XLSX` 或类似表达时，额外生成：

```text
项目能力表.xlsx
```

XLSX 是 `项目能力表.csv` 的阅读层，不是第二份事实源。先稳定 CSV，再从同一组行数据生成 XLSX，避免 Markdown、CSV 和 XLSX 之间内容漂移。

## 工作簿结构

默认包含 4 个工作表：

| 工作表 | 用途 |
|---|---|
| `阅读说明` | 说明项目名、文件用途、证据来源、后续能力标记规则、验证命令和阅读注意事项。 |
| `状态汇总` | 汇总 L3 功能点总数、L1 能力域数量、L2 功能组数量、各状态数量和状态标签含义。 |
| `能力表` | 主表，必须与 `项目能力表.csv` 同表头、同行数、同顺序。 |
| `L1-L2索引` | 按 L1 / L2 聚合 L3 数量和 L2 下一步，帮助快速浏览能力结构。 |

## `能力表` 表头

必须与 CSV 保持一致：

```csv
一级能力域,二级功能组或模块,三级功能点,功能说明,证据路径,当前状态,下一步,备注
```

不要在主表增加完成率、评分、权重、排期、负责人、成本、资源、成熟度或工作分解字段。

## `阅读说明` 建议字段

| 字段 | 写法 |
|---|---|
| `文件用途` | 用一句话说明这个工作簿用于阅读项目当前能力与后续能力地图。 |
| `证据来源` | 列出真实代码、README、docs、local 材料、SQL/seed、测试脚本、git log、验证命令等。 |
| `后续项标记` | 说明来自规划材料的后续能力标为 `已规划`；已有 adapter / smoke 但缺 live 证据的标为 `待验证`；只有推断来源的内容标记 `[推断]`。 |
| `状态限制` | 说明不使用完成率、成熟度评分或排期字段。 |

## `状态汇总` 规则

至少统计：

- `L3 功能点总数`
- `L1 能力域数量`
- `L2 功能组数量`
- 每个状态标签的 L3 数量

状态标签说明必须覆盖：

- `已规划`
- `已设计`
- `已开发`
- `可运行`
- `待验证`
- `已废弃/重构`
- `未发现证据`

## `L1-L2索引` 表头

```csv
一级能力域,二级功能组或模块,L3 数量,说明
```

`说明` 默认使用该 L2 的下一步建议。不要在索引里引入新的事实判断。

## 格式规则

- 完全汉化：工作表名、标题、表头、状态说明、备注、阅读说明都使用简体中文。
- 代码路径、命令、API 名、配置键、包名、真实模块名保留原文。
- 使用冻结首行、筛选、自动换行和适合长路径阅读的列宽。
- 可以对 `当前状态` 使用稳定色块，便于筛选和阅读。
- 所有工作表使用统一字体。中文环境优先用 `PingFang SC`；跨平台交付可用 `Arial`。
- 如果输出目录已有旧 XLSX，默认新建带日期或短后缀的目录/文件，不直接覆盖，除非用户明确要求。

## 验证规则

- 用 `openpyxl` 重新打开工作簿，检查工作表名、表头、主表行数和 CSV 行数一致。
- 如果工作簿使用公式，按 spreadsheet / xlsx skill 要求重新计算并确认没有公式错误。
- 如果只是静态表，不必强行添加公式；优先保证内容一致、可筛选、可阅读。
