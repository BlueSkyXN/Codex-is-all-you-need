# Stage 1 — Known Knowns: Open with the Settled Ground

Open by separating settled facts from assumptions. The user should immediately
see what does not need more discussion and what still does.

## Scan first (silent)

Before asking the user anything, inspect the relevant local truth. Read the
likely files, docs, or logs; note existing behavior, abandoned attempts, data
shapes, naming conventions, and constraints that could steer the task.

## The opening reply

- State the request in your words.
- List confirmed facts with citations to the files or evidence that confirm
  them.
- Separate constraints from assumptions. Mark soft assumptions as "using this
  unless you correct it."
- Preview the remaining quadrants in one line each so the user knows what the
  walk will cover.
- If the scan found a fact that changes the task, disclose it now and record it
  under the right quadrant.

**Done when** the user can see confirmed facts, assumptions, and the next three
quadrants without having to infer them from scrollback.
