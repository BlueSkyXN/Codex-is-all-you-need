# 使用案例

## 1. 产品首页布局

用户：

```text
我们要重做 AI 客服产品首页，但团队对首屏方向没有共识。
```

建议过程：

1. 先确认目标访客和首要转化目标；
2. 将问题限定为首屏的信息入口；
3. 展示“证据优先 / 产品优先 / 叙事优先”；
4. 保持品牌、功能和 CTA 一致；
5. 读取选择；
6. 下一屏再处理导航或信息密度。

成品参考：`examples/01-product-layout.html`。

## 2. 模型路由

用户：

```text
把统一强模型、规则路由和大小模型级联三种方式画出来，我想看请求路径差异。
```

视觉页面应标注：

- 分类或路由节点；
- 快路径和升级路径；
- 工具、缓存或检索位置；
- 失败回退；
- 指标属于事实、估算还是示意。

成品参考：`examples/02-model-routing.html`。

## 3. 系统边界

用户：

```text
我们要在现有 SaaS 中加入文档问答，请比较内嵌式和独立服务式架构。
```

两张图应使用相同尺度，并至少展示：

- 上传入口；
- 解析与索引；
- 检索与推理；
- 租户隔离；
- 存储位置；
- 故障影响面；
- 跨服务接口。

不要凭空添加吞吐、价格或 SLA。

成品参考：`examples/03-system-architecture.html`。

## 4. 审批流程

用户：

```text
把内容发布审批画成两套流程：固定串行审批和风险分级审批。
```

两套流程都要展示：

- 提交；
- 审核节点；
- 退回；
- 拒绝；
- 超时或异常；
- 完成状态。

只改变分流策略，不隐藏某一方案的异常路径。

成品参考：`examples/04-process-flow.html`。

## 5. 不应触发

### 纯参数表

```text
比较三个模型的上下文长度、价格和发布日期。
```

若只需要事实参数，直接用文字或表格。只有进一步需要看模型路由、数据流或部署边界时才启用浏览器。

### 普通需求澄清

```text
这个需求包括批量导出吗？
```

直接文字回答。

### 代码调试

```text
解释这段 Python 为什么报错。
```

直接分析代码、日志和堆栈，不制作视觉伴侣页面。

### 隐藏推理

```text
展示 Codex 正在执行的每一步内部思考。
```

不能把本 Skill 描述为内部推理查看器。可以展示公开的任务计划、架构图、文件变更摘要或测试结果，但必须明确这些是生成的外部产物。

## 6. 边界样例

### “比较三个模型”

这句话本身不足以决定是否触发：

- 比较价格、上下文和发布日期 → Markdown 表格；
- 比较请求路由、工具链和回退路径 → 浏览器图；
- 比较延迟分布，但没有数据 → 先索要或查找数据，不编造图；
- 比较 UI 中如何选择模型 → 可做交互 mockup。

### “给我画个图”

用户明确要求图形，但仍需判断介质：

- 简单流程 → Mermaid 可能足够；
- 多套同尺度方案并排选择 → 本 Skill；
- 需要像素级成品图 → 专门图像或设计工具；
- 需要真实数据图表 → 数据分析和图表工具。

## 7. 推荐调用提示

### 通用调用

```text
$visual-brainstorming
只把真正依赖视觉判断的部分放进本地浏览器。每屏只比较一个变量，给出 2–4 个结构上真实不同的方案，记录我的点击选择，并把最终结论写回项目文档。
```

### UI

```text
$visual-brainstorming
针对 1440px 桌面控制台给出三套导航结构。内容、配色和功能保持一致，只改变导航；在浏览器中让我点选。
```

### 架构

```text
$visual-brainstorming
用同一尺度比较“主应用内嵌检索”和“独立 RAG 服务”。不要编造性能数据，将所有非实测数字标为示意或估算。
```

### 流程

```text
$visual-brainstorming
比较固定串行审批与风险分级审批。两套方案都要显示退回、拒绝、超时和异常升级路径。
```

### 模型路由

```text
$visual-brainstorming
画出统一强模型、规则路由和置信度级联三种请求路径。只比较结构和治理代价，不引用未经验证的模型指标。
```

## 8. 一次完整对话示例

### 用户

```text
我们的后台导航越来越乱，做几个方向给我看。
```

### Agent

先确认：主要用户、模块数量、最常见任务和目标屏幕宽度。随后定义：

```text
本屏只决定导航结构；内容、配色、权限和功能不变。
```

若用户尚未明确要求浏览器，询问一次是否启用。得到同意后：

1. 启动伴侣；
2. 基于 `assets/templates/ui-compare.html` 生成三套结构；
3. 发布并结束当前轮次；
4. 下一轮读取事件；
5. 复述选择；
6. 再处理密度或折叠规则；
7. 把结论写入项目设计文档；
8. 停止服务。
