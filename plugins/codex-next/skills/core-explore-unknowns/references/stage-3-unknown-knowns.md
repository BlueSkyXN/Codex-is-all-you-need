# Stage 3 - Unknown Knowns: Surface Implicit Context

This stage extracts context that usually appears too late: taste, vocabulary,
consumer expectations, deployment assumptions, and local conventions that nobody
thought to mention.

## Procedure

1. Give the user a concrete artifact to react to: sample output, option table,
   prototype, vocabulary ladder, or shape sketch.
2. Ask targeted context probes the user may not volunteer: who consumes this,
   where it runs, what counts as done, what must remain compatible, and what
   would make the result feel wrong.
3. If a newly surfaced fact changes an earlier decision, update the map rather
   than preserving the old path for appearances.

**Done when** the user has reacted to a concrete artifact and the extracted
tacit context is captured or explicitly marked unknown.

## Techniques

- **Alternative drafts**: show two or more incompatible directions over the same
  realistic input so preference becomes visible.
- **Disposable mock**: build only enough interaction to answer taste or workflow
  questions before touching production code.
- **Vocabulary bridge**: translate the user's wording into the domain terms that
  unlock precise requirements.
- **Sample artifact**: for APIs, reports, files, or configs, show the actual
  shape with labeled example data and editable choices.
